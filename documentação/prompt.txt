Eu sou um Product Manager e você é um programador PYTHON e vai mexer com um grande banco de dados públicos da Receita Federal do Brasil, com arquivos que ultrapassam 1gigabyte de informação. Nesse primeiro momento faremos o sistema em uma máquina local, em ambiente preparado para isso (localhost), mas no futuro subiremos ele para a nuvem. Os dados estão disponíveis em vários arquivos modelo RDBMS com separação por ponto e vírgula e com diversas informações, conforme arquivo meta-dados anexo. Vamos criar um script para encontrar  empresas, estabelecimentos, dados do simples, sócios, países, municípios, qualificações de sócios, naturezas jurídicas e CNAEs. O usuário terá uma tela para fazer seus filtros e receberá as informações paginadas e poderá exportar o resultado para um arquivo excel. O principal dado que conecta as informações dos arquivos, unique, é o CNPJ raiz. A partir dele teremos os estabelecimentos. Cada CNPJ tem no mínimo 1 estabelecimento e até 9998 outros. O radical do CNPJ é composto por  8 números, como no exemplo: 00.000.000, em seguida vem o código do estabelecimento sendo o 0001 o principal/matriz e a partir daí as filiais e subsidiárias, e por fim um código verificador, formando um número de 14 caracteres assim: 00.000.000/0001-91. Os sócios no estabelecimento  0001 estão nas subsidiárias, que poderão ter outros sócios. Existe um cálculo para o código verificador com base no radical e no estabelecimento, formando o código verificador de dois dígitos (final). Ver como calcular o DV (Dígito de Verificação) do CNPJ no arquivo RegraDVdoCNPJ.txt anexo. As informações são atualizadas todos os meses, uma vez ao mês. Vamos manter histórico do que for alterado, salvando a data de inclusão da informação no bd e de sua atualização. Assim o usuário poderá ter acesso aos sócios de um CNPJ em diferentes tempos, se houver mudança no Quadro Societário.

Vamos dividir o trabalho em 2 fases:
1 - escolha dos frameworks e tecnologias e ordem de desenvolvimento;
2 - desenvolvimento dos scripts para importação dos dados públicos no sistema e disponibilização para o usuário (momento em que daremos mais informações sobre as necessidades do usuário);

As regras para a fase 1 é utilizar python como backend e postgreeSQL como banco de dados. Lembre-se que como iremos trabalhar com muitas informações, tanto para busca como para validação das mesmas no banco de dados, precisamos pensar na otimização da busca e inserção dos dados no bd.
